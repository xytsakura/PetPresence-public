# Media And Data Policy

PetPresence is local-first. Pet photos, videos, extracted frames, event logs, and generated reports can contain private information. Treat them as private by default.

## Default Rule

Do not commit user-specific media or generated observation data unless it is intentionally prepared as a public demo fixture.

Private by default:

- raw pet photos and videos;
- generated video-model outputs;
- foreground extraction previews;
- extracted observer frames;
- event JSONL files;
- generated daily reports;
- `outputs/` render artifacts;
- temporary smoke pet directories.

## Public Demo Fixtures

Public demo fixtures must satisfy all of these:

- the project has the right to publish them;
- they do not expose private homes, people, addresses, account names, keys, or provider metadata;
- they are small enough for a normal Git clone;
- they are referenced by docs or tests;
- their purpose is clear to a new contributor.

If a fixture is only useful for a one-time hackathon presentation, keep it out of the public repository or move it to an archived release asset.

## Current Repository Reality

This repository may contain historical hackathon demo assets and reports in a local working copy. They are excluded from the public release surface unless their publication rights are explicitly confirmed.

Before switching the repository to public, review at least:

```text
assets/pets/pet_demo/
data/pets/pet_demo/
assets/pets/pet_bichon_demo/
data/pets/pet_bichon_demo/

```

## Git Ignore Policy

`.gitignore` blocks future local generated outputs such as:

- `outputs/`;
- `data/pets/*/events/`;
- `data/pets/*/frames/`;
- `data/pets/*/reports/`;
- alpha preview/contact-sheet images generated by the conversion pipeline.
- legacy raw `private legacy media paths`, `private legacy data paths`, `shelter/adoption demo media/`, and `shelter/adoption demo data` local materials.

This protects new user workspaces. It does not remove files already tracked by Git.

## Agent Guidance

When helping a user create a pet:

1. Ask before using external model providers.
2. Never commit the user's private media by default.
3. Keep new pet IDs local unless the user explicitly wants to publish them.
4. Prefer ASCII `pet_id` values.
5. Run `pet:validate` before previewing.
6. Mention which generated files are private and should not be pushed.
